## 一、检测技术原理

### 1.1 频域分析（FFT）

AI 生成图片（尤其是 GAN 产出）在频率空间呈现出真实相机图片所没有的特定模式。快速傅里叶变换（FFT）将图像转换到频域，寻找肉眼不可见的光谱伪影。

**适用场景**：图片检测
**优势**：不依赖元数据，即使元数据被剥离仍有效
**局限**：对高度压缩的图片效果下降

### 1.2 EXIF 元数据检查

相机拍摄的照片嵌入元数据——相机型号、GPS、时间戳、镜头信息。AI 生成图片通常完全没有这些元数据，或包含与声称来源不符的元数据。部分工具（Adobe Firefly、C2PA 兼容生成器）在 EXIF/IPTC 字段中嵌入 AI 来源水印。

**适用场景**：图片检测
**优势**：快速、确定性高（有水印时）
**局限**：元数据可被手动篡改或删除

### 1.3 GAN 与扩散模型伪影分类器

在大量真实和 AI 生成图片上训练的神经分类器，学习识别细微视觉伪影——过度平滑的纹理、不一致的光照物理、不自然的眼睛高光、扩散模型在头发和手指等细节区域的模糊。

**适用场景**：图片检测
**准确率**：85-98%（取决于生成器类型）
**局限**：随着生成器改进和多样化，检测器准确率会下降

### 1.4 时序一致性分析（视频）

AI 生成视频的帧与帧之间存在微观不自然过渡——亮度差异、面部表情不自然、重复组件等。通过面部标志检测、时序一致性分析、音视频相关性检查来识别。

**适用场景**：视频检测
**准确率**：80-96%
**局限**：对高级生成器（如 Veo 3）准确率下降至 95% 左右

### 1.5 生成器指纹识别

每个 AI 生成工具都有独特的 UMAP 聚类特征。即使伪造内容在视觉上完美无缺，其指纹也能表明来源工具。

**适用场景**：图片+视频
**优势**：可识别具体生成工具（如 Grok Aurora、Midjourney v7）
**局限**：需要持续更新指纹库

## 二、开源方案详解

### 2.1 WeAreTheArtMakers/AIDetector

- **GitHub**: https://github.com/WeAreTheArtMakers/AIDetector
- **许可证**: 开源
- **技术栈**: Python 3.8+，4GB+ RAM（推荐 8GB）
- **检测方法**: 多引擎集成（扩散指纹 + 相机证据 + 文本取证 + 生成器指纹 + 提示词恢复）
- **准确率**:
  - T2I（Midjourney/DALL-E/SD）: 85-95%
  - 照片级 AI（Grok/Gemini）: 75-90%
  - 真实照片: 90-95%
  - AI 文本伪影: 70-85%
  - 合成/拼接: 80-90%
- **API**: RESTful，POST /analyze
- **特色**: 返回 AI 概率分数、置信度、证据级别、生成器指纹匹配、提示词重建

### 2.2 ACE-deepfake-detector

- **GitHub**: https://github.com/Anyashprasad/ACE-deepfake-detector
- **版本**: ACE v2.4
- **技术**: Xception 架构 + 迁移学习 + 混合精度
- **训练数据**: 140K 真假面孔 + FaceForensics++
- **功能**: 图片和视频深度伪造检测
- **特色**: 混合池化（视频一致性）、自适应阈值偏移、鲁棒面部提取
- **API**: `predict_image()`, `predict_video()`

### 2.3 pratikrath126/ai-image-detector

- **GitHub**: https://github.com/pratikrath126/ai-image-detector
- **在线 Demo**: https://aiimagedetector-ten.vercel.app
- **技术栈**: Next.js 14 + TypeScript
- **检测引擎**: 三引擎集成
  1. Sightengine GenAI（商业级）
  2. Ateeqq Space ViT（Hugging Face 上的 Vision Transformer）
  3. 本地元数据分析引擎
- **特色**: 加权投票系统、赛博朋克 UI、隐私优先（不存储图片）
- **部署**: Vercel 一键部署

### 2.4 Detector Conan

- **GitHub**: https://github.com/tahangz/Detector_Conan_for_deepfakes
- **技术栈**: MERN（MongoDB + Express + React + Node）+ Django AI API
- **模型**:
  - 图片: XceptionNet + ViT
  - 视频: ResNeXt-50 + LSTM
- **准确率**: ~92%
- **功能**: 用户认证、检测历史、仪表盘、个人资料管理、深色/浅色模式

### 2.5 中科院 Deepfake Defenders

- **开发方**: 中国科学院自动化研究所 VisionRush 团队
- **技术**: CNN 特征提取 + 异常检测 + GAN 对抗训练 + 多模态分析
- **功能**: 像素级分析、实时识别、开源协作
- **应用场景**: 社交媒体监控、新闻验证

### 2.6 IVY-FAKE / IVY-XDETECTOR

- **来源**: π3 AI Lab + 武汉大学 + 南京大学 + 斯坦福大学
- **论文**: https://arxiv.org/abs/2506.00979
- **数据集**: 15万+训练样本（94,781张图片 + 54,967个视频）
- **特色**: 统一图像+视频检测 + 可解释性（自然语言解释判断依据）
- **准确率**:
  - GenImage 基准: 98.36%（之前最佳 86.88%）
  - GenVideo 基准: >99% F1
  - Chameleon 基准: 85.20%（之前最佳 65.77%）

- **隐私**: 零数据保留，不存储提交内容

## 五、技术选型建议

### 第一阶段（MVP）

**推荐架构**：
```
用户 → Next.js 前端（Vercel）→ FastAPI 后端（Railway）
                                    ├→ DeepFlag API（图片，免费）
                                    ├→ Hugging Face Space（图片，ViT模型）
                                    └→ 本地元数据分析引擎
```

**关键决策**：
- 图片检测优先，视频第二阶段加
- 用免费 API 做主力，开源模型做备份和增强
- 前端用 Next.js（生态成熟、部署简单）
- 后端用 Python/FastAPI（AI 生态友好）

### 技术风险

1. **军备竞赛风险**：生成器持续进化，检测器需要持续更新
2. **误报风险**：对真实照片的误判会影响用户体验
3. **合规风险**：不同地区对 AI 检测的法律要求不同
4. **API 依赖风险**：免费 API 可能随时变更政策

### 风险缓解

1. 多引擎集成 + 加权投票，降低单一引擎失效风险
2. 自部署核心模型作为保底方案
3. 明确标注"概率参考"而非"确定性判断"
4. 持续关注 C2PA 等行业标准
