TrueLens · Polar 后台配置指引

把代码里已完成的 Polar MoR 收款链路（commit 739c9c9）从「待激活」变成「线上可收款」。照着点即可，预计 20 分钟。
⚠️ 先决定：Sandbox 还是 Production？（头号坑，先看这里） Polar 有 Sandbox（测试） 和 Production（生产） 两套完全隔离的环境。产品、Access Token、Webhook Secret 在两边各自独立——你在哪边建的产品，就必须用哪边的 Token 和 Secret，且 POLAR_ENV 要对上：
  • 产品 ID：sandbox 建的产品 ID ≠ production 建的产品 ID，不能混用。
  • Access Token / Webhook Secret：两套环境的令牌不同，必须同环境配对。
  • POLAR_ENV：填 sandbox 还是 production，要和上面三者保持一致。
强烈建议顺序：先在 Sandbox 建一套测试产品 + 测试凭据，把「付款 → Webhook → 到账额度/套餐 → 重试不重复发放」整条链路跑通（不花真钱），确认无误后再切到 Production 用真实产品/凭据上线。四者环境不一致是 403 / 找不到产品 的头号原因。
这份指引在做什么？ TrueLens 已经写好调用 Polar 的代码，但还没有密钥和产品 ID。你需要去 Polar 后台：① 建 3 个产品、② 生成 Access Token、③ 配 Webhook，然后把这些信息填进 .env.local 并部署。做完，用户点「升级」就会跳到 Polar 收银台，付完钱 Webhook 自动把额度/套餐写回数据库。

链路回顾（点升级后发生什么）

定价页点「升级」 ─POST /api/checkout {productKey}─▶
   若 POLAR_* 已配：createCheckout() 跳 Polar 托管收银台
   若未配：退回 mock 直接发额度（演示用）
Polar 收款成功 ─Webhook─▶ /api/webhooks/polar ─验签─▶ grantCredits / set plan
   （WebhookEvent 表防重复发放）

涉及到的环境变量与产品 key 全部见下方步骤。

第 1 步 · 注册 Polar 并建组织

1

打开 https://polar.sh，用 GitHub 或邮箱注册登录。

登录后第一时间创建一个 Organization（组织）——产品、Token、Webhook 都挂在组织下，不是挂在个人账号下。

别忘了收款入驻（Payout Onboarding） Polar 是 Merchant of Record，它通过 Stripe Connect Express 把款项打给你。建完组织后，去 Settings → Payouts / Onboarding 填好个人或公司收款信息，否则钱收了也提不出来。中国个人通常可入驻。

第 2 步 · 生成 Access Token

2

组织后台 → Settings → Access Tokens → Create Token。

  • 起个名字（如 truelens-prod），权限给默认的 Full access 即可。
  • 点创建后只会显示一次这串 polar_pat_... 令牌，立刻复制。

把令牌填进 .env.local 的：

POLAR_ACCESS_TOKEN=polar_pat_xxxxxxxxxxxxxxxx

第 3 步 · 建 3 个产品

3

组织后台 → Products → New Product。按下面三行分别建 3 个产品：

产品 key类型建议定价填到哪个环境变量
pro Subscription（订阅，按月） ¥39/月 或 $5.99/月 POLAR_PRO_PRODUCT_ID
business Subscription（订阅，按月） ¥199/月 或 $29/月 POLAR_BUSINESS_PRODUCT_ID
addon One-time（一次性） ¥19 或 $2.99 POLAR_ADDON_PRODUCT_ID
⚠️ 复制的是「Product ID」，不是 Price ID！ 每个产品建好后，列表上点 ⋯ → Copy Product ID（形如 xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx），把它填进对应的环境变量。代码里 createCheckout 用的是 products:[productId]，拿错 ID 会导致「找不到产品」报错。

价格/币种按你的出海策略填即可——代码只认 Product ID，不关心金额和币种。若想让定价页也显示美元，那是另一项「海外版定价页」改造，不在本指引范围。

第 4 步 · 配置 Webhook

4

组织后台 → Settings → Webhooks → Add Endpoint。

  • Endpoint URL：填你线上域名下的接口（把 truelens.top 换成你实际部署的域名，含不含 www 前后要一致）：
https://truelens.top/api/webhooks/polar
  • 订阅事件：勾选以下 5 个（代码只处理这几个，多勾无害）：
order.paid
subscription.created
subscription.active
subscription.canceled
subscription.revoked
  • 创建后页面会显示一个 Webhook Secret（whsec_...），复制它填进：
POLAR_WEBHOOK_SECRET=whsec_xxxxxxxxxxxxxxxx
Sandbox 与 Production 是两套隔离环境 Polar 的沙盒（测试）和生产各有一套独立的产品和独立的 Webhook Secret。建议先在 Sandbox 用测试产品跑通（POLAR_ENV=sandbox + 沙盒的产品 ID + 沙盒的 secret），确认无误再切到 Production 填真实产品 ID 和真实 secret、并把 POLAR_ENV 改为 production。

第 5 步 · 填 .env.local

5

把上面拿到的 6 个值，连同基础地址，写进项目根目录的 .env.local（参考 .env.local.example）：

# Polar.sh（出海收款 MoR）
POLAR_ACCESS_TOKEN=polar_pat_xxxx
POLAR_WEBHOOK_SECRET=whsec_xxxx
POLAR_ENV=sandbox          # 上线前改 production
POLAR_PRO_PRODUCT_ID=xxxx-xxxx-xxxx-xxxx
POLAR_BUSINESS_PRODUCT_ID=xxxx-xxxx-xxxx-xxxx
POLAR_ADDON_PRODUCT_ID=xxxx-xxxx-xxxx-xxxx
NEXT_PUBLIC_BASE_URL=https://truelens.top   # 付款成功后的回跳地址前缀，须与部署域名一致

只要 POLAR_ACCESS_TOKEN 和 POLAR_WEBHOOK_SECRET 两个都填上，polarEnabled() 就返回 true，/api/checkout 会自动从 mock 切到真实 Polar 收银台。

第 6 步 · 建表（新增 WebhookEvent）

6

这次集成新增了 webhook_events 表（幂等防重复发放）。需要把 schema 推到数据库：

# Windows cmd（URL 不要加引号）
set DATABASE_URL=postgresql://user:pass@host:5432/db?sslmode=require
npx prisma db push

Git Bash 下则为 DATABASE_URL='...' npx prisma db push。若用 scripts/db-push.sh 也行，但 Windows 下直接跑上面的命令更稳。命令会新建 webhook_events(id, type, createdAt) 表，已有的 users 表不改动。

第 7 步 · 部署上线

7

把 .env.local 里的 Polar 变量同步到 Vercel 的环境变量（Project → Settings → Environment Variables），然后部署：

vercel --prod --force

部署完成后，打开线上站点滚到页脚，应能看到版本号带新的 commit 短哈希（739c9c9 附近），确认新代码已生效。

第 8 步（推荐）· 沙盒端到端测试

8

在切 production 前，用 Sandbox 跑一遍真流程：

  • 临时把 POLAR_ENV=sandbox + 沙盒产品 ID + 沙盒 secret，部署一个预览环境（或本地带 tunnuel）。
  • 登录你的测试账号 → 定价页点「升级 Pro」→ 应跳到 Polar 沙盒收银台。
  • 用 Polar 沙盒提供的测试卡完成支付 → 回到 /pricing?polar=success。
  • 查数据库该用户的 plan 应变 pro、credits 增加；webhook_events 表里应多一条 order.paid 或 subscription.active 记录。
  • 故意让 Webhook 重试一次（Polar 后台可手动 Redeliver）→ 确认额度没有重复发放（幂等生效）。

常见问题 / 排错

点升级直接发额度、没跳 Polar 收银台

说明 polarEnabled() 为 false——POLAR_ACCESS_TOKEN 或 POLAR_WEBHOOK_SECRET 至少有一个没填上/没部署到 Vercel。检查环境变量是否同步到了 Vercel。

Webhook 返回 403 invalid_signature

Secret 不对或环境错配。确认：① POLAR_WEBHOOK_SECRET 是从当前环境（sandbox/production）复制的 secret；② 没在字符串前后多粘空格或换行。

付款成功但套餐没变

① 先查 webhook_events 表有没有这条事件——有但套餐没变，看 Vercel 函数日志里 [polar] 报错；② 确认产品 ID 复制的是 Product ID 而非 Price ID（resolveProductKey 靠它匹配 pro/business/addon）；③ 确认 Webhook 订阅了 order.paid / subscription.active。

successUrl 回跳 404 或域名不对

NEXT_PUBLIC_BASE_URL 必须和实际部署域名一致（含/不含 www 保持一致），它决定付款后跳回 {BASE}/pricing?polar=success。
