# 商业模式设计

> TrueLens - AI 内容真伪检测平台
> 制定日期：2026年7月16日

---

## 一、价值主张

**对个人用户**：一键辨别图片/视频真伪，不再被 AI 生成内容误导

**对企业用户**：API 接入，批量检测，保护平台内容生态和品牌信任

**核心差异化**：
- 透明可解释——不只是给一个分数，还告诉用户为什么
- 多引擎集成——降低单一检测器的误判风险
- 免费起步——降低用户尝试门槛

## 三、收入模型

### 3.1 订阅制（SaaS）

| 层级 | 月费 | 年费 | 功能 |
|------|------|------|------|
| Free | ¥0 | ¥0 | 1次/天，图片检测，基础报告 |
| Pro | ¥39 | ¥390（省1个月） | 无限检测，图片+视频，详细报告，历史导出 |
| Business | ¥199 | ¥1,990 | API 10,000次/月，批量检测，团队协作 |
| Enterprise | 定制 | 定制 | 私有部署，定制模型，SLA，专属支持 |

### 3.2 API 按量计费

| 调用量 | 单价 |
|--------|------|
| 1-1,000 次 | ¥0.5/次 |
| 1,001-10,000 次 | ¥0.3/次 |
| 10,001-100,000 次 | ¥0.15/次 |
| 100,000+ 次 | 定制报价 |

### 3.3 增值服务

- **定制检测报告**：¥9.9/份（PDF，含详细证据链）
- **企业定制模型**：针对特定生成器训练专用检测器
- **培训与咨询**：AI 内容识别培训课程
- **白标方案**：为其他平台提供底层检测能力

## 四、成本结构

### 4.1 固定成本（月）

| 项目 | MVP阶段 | 成长期 | 规模化 |
|------|---------|--------|--------|
| 服务器 | ¥0 | ¥300 | ¥1,500 |
| 域名 | ¥5 | ¥5 | ¥5 |
| 支付网关 | ¥0 | ¥0 | ¥0（按交易抽成） |
| **合计** | **¥5** | **¥305** | **¥1,505** |

### 4.2 变动成本

| 项目 | 计算方式 |
|------|----------|
| 商业 API 调用 | 按实际调用量付费 |
| 服务器扩容 | 按流量自动扩容 |
| CDN 流量 | 按使用量 |
| 支付手续费 | 交易额 2-3% |

### 4.3 盈亏平衡分析

假设：
- Pro 用户占比 70%，Business 占 25%，Enterprise 占 5%
- 月活用户 2,000 人
- 付费转化率 5%（100 付费用户）

月收入估算：
- Pro: 70 × ¥39 = ¥2,730
- Business: 25 × ¥199 = ¥4,975
- Enterprise: 5 × ¥1,000（估） = ¥5,000
- API: ~¥2,000
- **合计：~¥14,705**

月成本估算：
- 服务器+API: ~¥2,000
- 其他: ~¥500
- **合计：~¥2,500**

**月利润：~¥12,205**

**盈亏平衡点：约 15-20 个付费用户**

## 五、竞争优势

### 5.1 成本优势

- 开源模型 + 免费 API 起步，成本极低
- OPC 模式无人员成本
- 自动化运营降低边际成本

### 5.2 技术优势

- 多引擎集成降低误判
- 可解释性输出（自然语言解释）
- 持续更新检测器应对新型生成器

### 5.3 模式优势

- 人类 + AI 合伙，决策效率高
- 轻资产运营，试错成本低
- 快速迭代，贴近用户需求

### 5.4 潜在竞争壁垒

- 检测准确率数据积累（用户反馈标注）
- 生成器指纹库持续扩充
- 品牌信任度（在内容真实性领域建立权威）

## 六、风险与应对

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| 生成器进化导致检测失效 | 高 | 高 | 持续更新模型，多引擎冗余 |
| 大厂免费提供类似功能 | 中 | 高 | 差异化定位（可解释性、垂直场景） |
| 法律合规要求变化 | 中 | 中 | 持续关注法规，明确免责声明 |
| 免费 API 政策变更 | 中 | 中 | 自部署核心模型作为保底 |
| 用户付费意愿低 | 中 | 高 | 先免费引流，用增值服务变现 |

---

## 七、退出策略

1. **持续经营**：作为 OPC 长期运营，月利润 ¥1-3 万
2. **被收购**：积累用户和检测数据后，被内容平台/安全公司收购
3. **转型 B2B**：聚焦 API 服务，成为底层检测基础设施
