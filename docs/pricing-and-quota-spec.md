# TrueLens 定价与额度护栏规格 v1

> 目的：一份可直接交给支付接入 / 配额系统实现的规格。核心原则——**信用点（credits）先付后耗，数学上不可能亏损**；高精度引擎（下称"高精度辨识"，对外不点名供应商）成本被三道护栏钉死。
> 状态：Michael 已拍板（免费高精度 3 次/月、视频:图片 = 8:1）。待工程实现 + 上线前接支付。

## 二、信用点（Credits）体系

**定义**：1 credit = 1 张图片的高精度辨识。视频按 **8 credits** 计（= 8 帧）。

| 动作 | 消耗 credits |
|---|---|
| 图片 · 高精度辨识 | 1 |
| 视频 · 高精度辨识（≤60s, 8 帧） | 8 |
| 图片 · 基础引擎（免费兜底） | 0（不计费） |

> 关键：credits 是**预购/月度发放**的额度。用户用完 = 用户额度耗尽（回落基础引擎或提示升级），**不是我们欠供应商钱** → 额度完全可控。

## 四、毛利测算（保守）

假设成本按 Starter $29≈¥210/月、10,000 ops：

- **纯图片场景**：500 个 Pro 用户 × 20 图/月 = 10,000 ops = 打满 Starter。
  收入 500×¥39 = **¥19,500**，成本 **¥210** → 毛利 **~98.9%**。
- **含视频的重度用户是唯一击穿点**：1 个 Pro 用户月传 200 视频 = 1,600 ops ≈ ¥33.6 成本，而 Pro 仅 ¥39 → 该用户毛利仅 ~14%。
  → 由护栏 3（视频高定价 8:1 + 数量随 credits 自然受限）兜底：Pro 500 credits 最多 ~62 个视频，成本 ¥8.3，仍安全。

**盈亏平衡**：覆盖 Starter ¥210 只需 **6 个 Pro 用户**；升级 Pro 档供应商（¥715）需 ~19 个付费用户。

## 五、工程侧护栏（必须实现，防超额/防亏损）

1. **全局月度 ops 预算闸** `MAX_SE_OPS_PER_MONTH`
   - 计数表记录当月已耗 ops；达 **80%** → 管理员告警；达 **100%** → 所有非管理员请求自动回落基础引擎，提示"高精度额度暂满，已切基础引擎"。
   - 防止供应商账单在 $0.002/op 超额下失控。

2. **免费/匿名默认不走高精度**
   - 路由到零成本基础引擎；每账户/设备每月仅 **3 次** 高精度体验额度，用完回落。
   - 判定在**服务端**（抓包也拿不到额外次数）。

3. **视频封顶**
   - 单视频时长 **≤60s**、帧数 **锁定 8**，把单次成本钉死在 8 ops。
   - 视频高精度仅对 Pro/Business 开放（Free 不开放）。

4. **按用户限速 + 单文件大小上限**
   - 遵守供应商 1 req/s（免费/Starter）；用户级速率限制防突发把额度打爆。
   - 图片 ≤10MB、视频 ≤50MB（示例，按实际调整）。

5. **credits 原子扣减**
   - 检测成功才扣 credits；失败/回落基础引擎不扣。DB 事务保证并发下不超发。
